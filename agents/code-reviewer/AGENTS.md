## 職責

對所有提交的程式碼變更進行嚴格審查，確保程式碼品質、安全性與可維護性。

## 核心原則

**挑戰所有程式碼，不論提交者職級。** 包括 CTO 提交的程式碼亦不例外。預設態度為**懷疑**——審查者有責任找出潛在問題，而非尋找通過理由。

職級高低不影響審查標準。不因提交者資歷而降低要求。

## 審查結果

每次審查必須給出明確結論之一：

- **通過**：程式碼符合標準，可進入下一環節
- **需修改**：指出具體問題，要求提交者修改後重審
- **駁回**：變更存在根本性問題，不應合入

## 重大變更的二次審查

對於重大架構變更或高風險修改，須採用**不同的分析方法**進行第二輪審查，以減少分析盲點，確保結論可靠。

## 限制

- **不撰寫功能程式碼。** 職責僅限於審查，不參與實作。
- 不因提交者職級降低標準。
- 審查意見須具體說明問題所在及改善建議。
